# Task 06 — CSV Export Capability

**Phase:** Build 4f  
**Owner:** Backend Dev  
**Priority:** Medium  
**Depends on:** Task 03 (MCP tools — `export_csv` stub), Task 05 (CLI `/export` command)

---

## Objective

Implement the full CSV export flow so that any query result set can be saved to a named CSV file from either the CLI or programmatically.

## Acceptance Criteria

- [ ] `export_csv` MCP tool (from Task 03) writes a valid CSV to `./exports/<auto-generated-name>.csv`.
- [ ] Auto-generated filename includes the file type, query parameters, and a timestamp (e.g., `medicaid_enrollment_2029_20260512_143022.csv`).
- [ ] The exported CSV includes all columns from the result DataFrame plus metadata headers (file type, vintage, export timestamp).
- [ ] CLI `/export` command triggers the export and prints the full path of the saved file.
- [ ] The `./exports/` directory is created automatically if it does not exist.
- [ ] Unit test in `tests/test_csv_export.py` verifies:
  - File is created at the returned path.
  - File is valid CSV (parseable by `pandas.read_csv`).
  - File contains at least the expected columns.
- [ ] `./exports/` is excluded from git (added to `.gitignore`).

## Implementation Notes

- Use `pandas.DataFrame.to_csv(index=False)` for output.
- Metadata headers can be written as comment lines (`# key: value`) before the data rows, or as a separate `_meta.json` sidecar file.
- Sanitize filename components to remove special characters.
